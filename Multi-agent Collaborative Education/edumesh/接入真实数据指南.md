# 📡 EduMesh 接入真实数据指南

## 一、接入真实 LLM (OpenAI / Anthropic)

### 1. 获取 API Key
- **OpenAI**: https://platform.openai.com/api-keys
- **Anthropic**: https://console.anthropic.com/settings/keys

### 2. 配置环境变量

```bash
# Linux/Mac
export LLM_PROVIDER=openai
export LLM_API_KEY=sk-your-openai-key-here
export LLM_MODEL=gpt-4

# Windows PowerShell
$env:LLM_PROVIDER="openai"
$env:LLM_API_KEY="sk-your-openai-key-here"
$env:LLM_MODEL="gpt-4"
```

### 3. 启动后端
```bash
cd backend
python main.py
```

> 代码已自动适配：LLMService 会根据 `LLM_PROVIDER` 自动选择 SDK，未安装则回退到 Mock 模式

## 二、接入真实数据库 (PostgreSQL / MySQL / SQLite)

### 方案 A: SQLite (最简单，单机测试)

```bash
# 无需安装额外依赖，已内置
export DATABASE_URL="sqlite:///edumesh.db"
python main.py
```

### 方案 B: PostgreSQL (生产推荐)

```bash
# 1. 安装 PostgreSQL
# Mac: brew install postgresql
# Ubuntu: sudo apt install postgresql

# 2. 创建数据库
createdb edumesh

# 3. 安装 Python 驱动
pip install psycopg2-binary

# 4. 配置连接
export DATABASE_URL="postgresql://user:password@localhost:5432/edumesh"

# 5. 初始化表 + 导入题库
python -c "from db_models import init_db; init_db()"
python import_questions.py default

# 6. 启动
python main.py
```

### 方案 C: MySQL

```bash
pip install pymysql
export DATABASE_URL="mysql+pymysql://user:password@localhost:3306/edumesh"
python main.py
```

## 四、修改 main.py 使用数据库 (关键步骤)

当前 `main.py` 使用的是内存存储 `students_db = {}`。要接入真实数据库，需要修改以下几个地方：

### 修改 1: 导入数据库模块

在 `main.py` 顶部添加：

```python
from db_adapter import db_store
from db_models import init_db

# 启动时初始化数据库
init_db()
```

### 修改 2: 替换学生存储操作

```python
# 原代码 (内存版)
students_db: Dict[str, StudentProfile] = {}

# 改为 (数据库版)
# students_db 不再需要，改用 db_store.get_student() / db_store.save_student()
```

### 修改 3: 修改 create_student

```python
@app.post("/api/students")
async def create_student(data: StudentCreate):
    import uuid
    sid = f"stu_{uuid.uuid4().hex[:8]}"
    student = StudentProfile(id=sid, name=data.name)

    # 保存到数据库
    db_store.save_student(student)

    # 初始化学习路径
    await curriculum_agent._build_learning_path(student)
    db_store.save_student(student)  # 保存路径

    return {"student_id": sid, "name": data.name}
```

### 修改 4: 修改 get_student / get_mastery

```python
@app.get("/api/students/{student_id}")
def get_student(student_id: str):
    student = db_store.get_student(student_id)  # 从数据库读取
    if not student:
        raise HTTPException(status_code=404, detail="学生不存在")
    return {
        "id": student.id,
        "name": student.name,
        "engagement_score": student.engagement_score,
        "total_questions": student.total_questions,
        # ...
    }
```

### 修改 5: Agent 中使用数据库

在 `assessment.py`, `curriculum.py` 等 Agent 中，将 `self.students[student_id]` 改为从数据库读取：

```python
# 在 Agent 的 __init__ 中
from db_adapter import db_store
self.db = db_store

# 在 handle 中
student = self.db.get_student(event.student_id)
if not student:
    return

# ... 处理逻辑 ...

# 处理完成后保存
self.db.save_student(student)
```

## 五、接入 Redis (分布式部署 / 缓存)

### 1. 安装 Redis
```bash
# Mac
brew install redis
brew services start redis

# Docker
docker run -d -p 6379:6379 redis:alpine
```

### 2. 安装 Python 依赖
```bash
pip install redis
```

### 3. 配置环境变量
```bash
export REDIS_URL="redis://localhost:6379/0"
```

### 4. 使用场景
- **分布式 EventBus**: 多个后端实例共享事件
- **Session 缓存**: 学生状态缓存，减少 DB 查询
- **Rate Limiting**: API 限流

## 六、生产环境部署

### Docker Compose (完整版)

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_USER: edumesh
      POSTGRES_PASSWORD: edumesh_pass
      POSTGRES_DB: edumesh
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    environment:
      - DATABASE_URL=postgresql://edumesh:edumesh_pass@postgres:5432/edumesh
      - LLM_PROVIDER=openai
      - LLM_API_KEY=${LLM_API_KEY}
      - REDIS_URL=redis://redis:6379/0
    depends_on:
      - postgres
      - redis
    ports:
      - "8000:8000"

  frontend:
    image: nginx:alpine
    volumes:
      - ./frontend:/usr/share/nginx/html:ro
    ports:
      - "3000:80"

volumes:
  pgdata:
```

### 启动
```bash
docker-compose up -d
```
